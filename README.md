# 📝 ToDo-лист: Чистая Архитектура (MVP)

> Минималистичный ToDo-лист, сфокусированный на демонстрации принципов чистой архитектуры и разделения ответственности в фронтенд-разработке.

## 🛠️ Технологии

*   **HTML5** — структура и семантика.
*   **CSS3 (Flexbox)** — стилизация и адаптивная верстка.
*   **JavaScript (ES6+)** — логика приложения.

## 🎯 О Проекте

Это не просто список задач, а учебный проект, демонстрирующий правильный подход к построению фронтенд-приложений. Главная цель — показать разделение кода на логические слои (UI, Controller, Core, Storage), что обеспечивает чистоту, масштабируемость и удобство поддержки.

## 🏗️ Архитектура (Ключевая особенность)

Проект построен по принципу разделения ответственностей (Separation of Concerns):

1.  **UI Layer (Отображение):**
    *   Отвечает только за отрисовку интерфейса. Не содержит бизнес-логики.
2.  **Controller Layer (Связь):**
    *   Переводит действия пользователя (клики, ввод) в команды для ядра приложения.
3.  **Core / Task Manager (Ядро/Бизнес-логика):**
    *   "Мозг" приложения. Хранит состояние (список задач) и содержит всю логику (добавить, удалить, отметить).
4.  **Storage Layer (Хранилище):**
    *   Абстракция для работы с данными. В текущей версии использует `localStorage`.

*Такая структура позволяет в будущем легко заменить `localStorage` на серверный API, не переписывая логику приложения.*

## 🚀 Как запустить локально

1.  **Скачайте** или склонируйте этот репозиторий.
2.  Откройте файл `index.html` в любом современном браузере.
